## Known Issues

### 1) Agent Subagent Depth Limited to 1

Copilot custom agents should be able to invoke skills and other agents according to the [Copilot agent tools documentation](https://github.com/github/copilot-cli/issues/839), but agent nesting depth is currently limited to 1. This means the `spec-mode` agent cannot spawn a discovery subagent to explore the codebase during the spec workflow. Workaround: run codebase exploration in the parent agent before delegating to `spec-mode`, or use `spec-mode` skills directly in default mode where the main agent has full tool access.

Related report: [easingthemes/dx-aem-flow#20](https://github.com/easingthemes/dx-aem-flow/issues/20)

### 2) Plan Mode and Todo Tool Compete With Spec Mode

Copilot's built-in plan mode and `#todo` tool can interfere with the spec workflow's own phase progression. If plan mode activates during a spec session, it may auto-advance phases or create parallel task tracking that conflicts with the spec's requirements → design → tasks gates.

Do this:
1. Do not use plan mode or `#todo` during spec workflow sessions.
2. After each phase, tell the agent what you want next — continue, revise, or generate remaining phases together.
3. If plan mode activates unexpectedly, tell the agent: `Do not use plan mode. Follow the spec workflow phase gates.`
