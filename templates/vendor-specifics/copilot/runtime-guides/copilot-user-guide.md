## Copilot Profiles

### Always-on

- Emits `.github/copilot-instructions.md`, `.github/instructions/*.instructions.md`, reusable skills, and this user guide.
- This is the primary Copilot profile currently implemented in full. It keeps the workflow inline instead of depending on generated Copilot agents.

### Lite

- Emits `.github/copilot-instructions.md` plus `.github/agents/*.agent.md`.
- The global instructions stay intentionally slim and route structured spec work into `spec-mode` to reduce context bloat.

### Agent-only

- Emits only `.github/agents/kiropen.agent.md` and `.github/agents/spec-mode.agent.md`.
- Use this when the team wants KirOpen only through explicit primary-agent selection, without repo-wide Copilot instructions.

## Known Issues

### Do Not Use Plan Mode; Use Todo As A Mirror When Available

Do not use Copilot's built-in plan mode to drive the spec workflow. If the current Copilot surface exposes `#todo` or another todo-capable tool, use it only as a progress mirror and not as a replacement for the workflow's explicit approval gates.

Do this:
1. In feature requirements-first sessions, track `Draft requirements` -> `Ask user for feedback` -> `Draft design` -> `Draft tasks`.
2. In technical design-first sessions, track `Draft design` -> `Ask user for feedback` -> `Draft requirements` -> `Draft tasks`.
3. If a todo-capable tool is available, keep exactly one phase todo active at a time, and do not move to the next phase until the user has given feedback or explicit approval.
4. For task execution, if a todo-capable tool is available, create one todo per task, keep the active task marked in progress, and mark completed work in both the todo list and `tasks.md`.
5. If Copilot starts using plan mode or todos in a conflicting way, tell it: `Do not use plan mode. Use todos only to mirror the spec workflow, not to replace the approval gates.`

### Lite Mode Intentionally Omits Full Repo Scaffolding

Lite mode does not emit path-scoped instructions, reusable skills, or runtime extras. That is intentional. If you later want repo-wide steering wrappers or reusable porting skills on disk, regenerate in Always-on mode instead of manually piecing those files together.

### Profile Maturity

Always-on is the most complete Copilot profile today. Lite and Agent-only remain available for review and narrower workflows, but the primary maintained path is Always-on.

## Limits

- This guide documents behavior; it does not change runtime policy.
- Runtime-level instructions may supersede repository templates.
- Copilot behavior varies across surfaces (VS Code, CLI, github.com coding agent). Not all features are available everywhere.
